# 実装後チェックポリシー

## 1. 目的

この文書は、ClaudeCodeによる実装後に、何をどう確認するかを事前に定義するためのものです。

## 2. チェックの基本方針

- 見た目だけでなく、情報設計と導線も確認する
- デザインだけでなく、初心者向けの分かりやすさも確認する
- 仕様書どおりかだけでなく、体験価値が伝わるかも確認する

## 3. 必須チェック項目

### 3-1. コンテンツチェック

- 各ページが想定見出し構成に沿っているか
- 初稿の意図が大きく崩れていないか
- 初心者にとって難しすぎる言い回しが増えていないか
- オプトアウト、セキュリティ、料金の説明が断定的すぎないか

### 3-2. デザインチェック

- `DESIGN.md` の方向性に沿っているか
- 体験セクションがしっかり目立っているか
- 解説セクションが読み疲れしないか
- 情報の強弱がついているか

### 3-3. 導線チェック

- トップから `まず体験してみる` へ自然に行けるか
- 各ページ末尾の次導線が機能しているか
- 迷った人向け導線が見つけやすいか

### 3-4. 図解チェック

- 図解が本文と一致しているか
- 1図1メッセージになっているか
- スマホでも読めるか
- 装飾だけで終わっていないか

### 3-5. レスポンシブチェック

- スマホで文字が小さすぎないか
- 表やカードが崩れていないか
- CTAが押しやすいか
- 図解がはみ出していないか

### 3-6. セキュリティ・品質チェック

- 不要な外部依存がないか
- 安全性に関する誤解を招く表現がないか
- 無駄なコードや未使用スタイルが多くないか

## 4. チェックの進め方

1. 要件と照らす
2. デザイン方針と照らす
3. 実機サイズで見る
4. 本文を実際に読む
5. 導線を順番にたどる

## 5. 合格条件

- 体験先行の価値が最初に伝わる
- 初心者でも読み進めやすい
- 主要導線と図解が崩れていない
- ルール文書に反する実装がない
