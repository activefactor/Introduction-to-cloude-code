Step 3 — 導入を選ぶ

導入方法を選ぶ

ClaudeCodeには3つの導入方法があります。
自分の環境や目的に合ったものを選んでみましょう。

このページで分かること

  • 3つの導入方法（Desktop / CLI / VS Code拡張）の違い
  • それぞれに向いている人・メリット・デメリット
  • 初心者におすすめの選び方
  • 迷ったときの判断フロー
3行まとめ

ClaudeCodeにはDesktopアプリ・CLIツール・VS Code拡張の3つの入口があります。
初心者にはDesktopアプリが一番始めやすく、CLI操作に慣れている人にはCLIが自由度高めです。
どれを選んでも後から変更できるので、迷ったら一番簡単そうなものから始めましょう。

3つの入口

導入方法は3つある

どれを選んでも同じClaudeCodeです。入口の違いだけです。

🖥️

Desktopアプリ

インストールして起動するだけ。設定が少なく、最もシンプルに始められます。

⌨️

CLI（コマンドライン）

ターミナルから直接呼び出す方法。自由度が高く、スクリプトとの組み合わせも得意。

🔌

VS Code + 拡張

エディタに組み込む方法。コードを書きながら隣にAIが使える環境が作れます。

詳しく比較する

それぞれの特徴を見る

🖥️

Desktopアプリ

初心者におすすめ

向いている人

CLIに慣れていない人、まず手軽に試したい人、設定を極力減らしたい人

メリット

  • インストールして起動するだけで使い始められる
  • GUIで操作しやすい
  • ターミナルの知識が不要
  • セットアップが最も少ない

デメリット（注意点）

  • CLI版と比べると細かい設定の自由度が低い場合がある
  • バージョンにより機能の差が生じることがある
⌨️

CLI（コマンドライン）

ターミナル慣れた人向け

向いている人

ターミナルを普段使う人、スクリプトと組み合わせたい人、細かいオプション設定をしたい人

メリット

  • 柔軟なオプション指定ができる
  • シェルスクリプトやCIとの連携がしやすい
  • 軽量で起動が速い
  • ファイル操作と自然に組み合わせられる

デメリット（注意点）

  • ターミナルの基礎知識が必要
  • 最初のセットアップに少し手間がかかる
  • CLI操作に慣れていないと最初は戸惑いやすい
🔌

VS Code + 拡張

開発者・エディタユーザー向け

向いている人

VS Codeをすでに使っている人、コードを書きながらAIに相談したい人

メリット

  • エディタから離れずにAIが使える
  • ファイルの参照・編集がスムーズ
  • VS Codeの操作感と自然に統合される
  • コーディング作業と組み合わせやすい

デメリット（注意点）

  • VS Codeのインストールと設定が前提
  • エディタに慣れていない人には向かない

3つの方法を並べて比べる

比較軸 Desktopアプリ CLI
始めやすさ ◎ インストールするだけ 〇 ターミナル操作が必要
自由度・柔軟性 〇 標準的な操作向け ◎ オプション指定など柔軟
必要な前提知識 特になし ターミナルの基礎
他ツールとの連携 △ 単体での利用が基本 ◎ スクリプト・CIと組みやすい
初心者のおすすめ度 ⭐⭐⭐ 最もおすすめ ⭐⭐ ターミナル慣れた人向け

※ VS Code拡張はVS Code利用者向けのため、上記には含めていません。

迷ったときに

迷ったときの選び方

いくつかの質問に答えると、自分に合う方法が見えてきます。

ターミナル（コマンドプロンプト）を普段使いますか？

No / 分からない → まず Desktopアプリ がおすすめ
Yes → 次の質問へ

VS Codeをエディタとして使っていますか？

Yes → VS Code拡張 が使いやすい
No → CLI が最も柔軟に使える

どれを選んでもOKです

補足 後から別の方法に切り替えることもできます。まず1つ試してみましょう。
結局、最初はどれがおすすめ？

ターミナルに慣れていない人は Desktopアプリ が一番始めやすいです。設定が少なく、すぐ始められます。
どの方法を選んでも、基本的な使い方は同じです。
