# ClaudeCode初心者向けオンボーディングサイト サイトマップ案

## 1. サイト全体の考え方

本サイトは、ClaudeCodeを初めて知る人が「全体像を知る」「自分に合う導入方法を選ぶ」「安心して使い始める」まで進める構成を目指します。

情報の流れは、次の3段階を基本とします。

1. まず感動する
2. 理解しながら試す
3. 安心して続ける

## 2. サイトマップ

```text
トップページ
├─ まず体験してみる
├─ ClaudeCodeとは
├─ Chat AIとの違い
├─ 導入方法を選ぶ
│  ├─ Desktopアプリ
│  ├─ CLI
│  └─ VS Code + 拡張
├─ はじめての使い方
├─ 便利な活用事例
├─ 料金とトークン
├─ モデルの使い分け
├─ セッション・記憶・ルール・スキル
├─ セキュリティ・オプトアウト・サンドボックス
├─ 用語集
└─ FAQ
```

## 5. 各ページの役割

### 5-1. トップページ

役割:

- サイト全体の入口
- ClaudeCodeの価値を短時間で理解してもらう
- まず驚きと興味を作る
- 読者の目的に応じた導線を提示する

### 5-2. まず体験してみる

役割:

- ClaudeCodeがファイルを作る・編集する体験を先に見せる
- 「これは普通のChat AIと違う」と実感してもらう
- その後の解説ページを読みたくなる状態をつくる

### 5-3. ClaudeCodeとは

役割:

- ClaudeCodeの基本イメージをつかんでもらう
- 「チャットAIの延長」ではなく「作業に関われるAI」と理解してもらう

### 5-4. Chat AIとの違い

役割:

- 既存のChat AI利用者にとっての違いを腹落ちさせる
- 実ファイルを扱えることの価値を具体的に示す

### 5-5. 導入方法を選ぶ

役割:

- Desktop、CLI、VS Code拡張の違いを比較し、自分に合う導入方法を選んでもらう

### 5-6. はじめての使い方

役割:

- 初回に何をどう試せばよいかを明確にする
- 最初の成功体験につなげる

### 5-7. 便利な活用事例

役割:

- 「なぜ便利か」を実例で理解してもらう
- 従来のやり方との比較で価値を示す

### 5-8. 料金とトークン

役割:

- 課金や制限への不安を減らす
- トークンという概念を必要十分に理解してもらう

### 5-9. モデルの使い分け

役割:

- 初心者がモデル選択で迷いすぎないようにする
- 作業内容ごとの選び方を示す

### 5-10. セッション・記憶・ルール・スキル

役割:

- ClaudeCodeならではの使い方を理解してもらう
- 継続利用で便利になる考え方を学んでもらう

### 5-11. セキュリティ・オプトアウト・サンドボックス

役割:

- 安全性の考え方をやさしく伝える
- 過度に怖がらず、適切に注意できる状態をつくる

### 5-12. 用語集

役割:

- 本文で出てきた用語を補助的に理解できるようにする
- つまずきを減らす

### 5-13. FAQ

役割:

- 読者の不安を最後に解消する
- よくある疑問への短く明快な答えを用意する

## 6. トップページの情報設計案

トップページでは、以下の順に見せると分かりやすいです。

1. ヒーロー
2. まず体験してみる導線
3. ClaudeCodeを一言で説明
4. Chat AIとの違いを一目で比較
5. 自分に合う導入方法の案内
6. 代表的な活用例
7. よくある不安への導線
8. おすすめの読み順

## 7. ページ共通のUI要件案

- 各ページ冒頭に「このページで分かること」を置く
- 各ページ末尾に「次に読むページ」を置く
- 用語は必要に応じて補足表示する
- 比較ページでは表形式を使う
- 長い説明は図解やカードに分解する
- 解説ページの次に、短い実践ステップページまたは実践セクションを置く

## 8. コンテンツのリズム設計

読者の集中を保つため、次のように解説と実践を交互に配置する構成を推奨します。

1. トップページで価値を知る
2. まず体験してみる
3. 違いを理解する
4. 導入方法を選ぶ
5. 実際に始める
6. 仕組みを理解する
7. 安全に使う
8. よくある疑問を解消する

## 9. 解説図の配置案

- トップページ: ClaudeCodeで起きることの全体図
- まず体験してみる: 指示からファイル作成までの流れ図
- Chat AIとの違い: 比較図
- はじめての使い方: ステップ図
- セッション・記憶・ルール・スキル: 関係図
- セキュリティ・オプトアウト・サンドボックス: 安全領域マップ

## 10. 更新しやすさを意識した構成案

変わりやすい情報と変わりにくい情報を分けて管理する想定です。

変わりやすい情報:

- 料金
- モデル
- トークン上限
- オプトアウトや設定に関する説明
- 導入手順

変わりにくい情報:

- Chat AIとの違いの考え方
- 活用事例の構造
- セッションやルールの基本概念
- サンドボックスの考え方
