# ClaudeCode初心者向けオンボーディングサイト トップページ ワイヤー案

## 1. このワイヤーの目的

トップページの役割は、ClaudeCodeをまだよく知らない人に対して、最初の30秒で「これは自分にも関係ありそう」「ちょっと体験してみたい」と思ってもらうことです。

そのため、構成は次の順番を基本にします。

1. まず惹きつける
2. 何がすごいかを短く伝える
3. 体験ページへ送る
4. 読者ごとの導線を分ける
5. 不安を先回りして下げる

## 3. セクションごとの設計

### 3-1. ヘッダー

目的:

- 全体の回遊性を作る
- 迷った人がすぐ別ページに行けるようにする

表示要素:

- サイト名
- ナビゲーション
- CTAボタン `まず体験してみる`

ナビ案:

- まず体験
- 導入方法
- 活用例
- 料金とモデル
- しくみを知る
- 安全に使う
- FAQ

### 3-2. ヒーロー

目的:

- 最初の感情をつくる
- ClaudeCodeの価値を一発で伝える

コピー案:

`会話するだけで終わらない。ClaudeCodeは、ファイルづくりまで一緒に進められるAI。`

サブコピー案:

`調べる・要約する・壁打ちする。その次の一歩として、実際にファイルを作ったり直したりできるのがClaudeCodeです。`

CTA案:

- 第一CTA: `まず体験してみる`
- 第二CTA: `ClaudeCodeとは？`

右側または下部に置く図:

- `指示 → 読み取り → ファイル作成/編集 → 確認` の4ステップ図

### 3-3. 3つの驚き

目的:

- ヒーローで気になった人に、価値を具体化して見せる

カード案:

1. `ファイルを作れる`
   説明: アイデアだけでなく、READMEやスクリプトのたたき台まで形にできる

2. `ファイルを直せる`
   説明: 修正案を話すだけでなく、実際の変更まで進められる

3. `作業を前に進められる`
   説明: 調査、整理、作成、改善を会話の流れでつなげられる

### 3-4. Chat AIとの違いミニ比較

目的:

- 既存のChat AI利用者にとっての違いを短時間で腹落ちさせる

表示方法:

- 2カラム比較
- 詳細ページへのリンク付き

比較項目:

- できること
- 実ファイルとの関わり
- 作業の進み方
- 向いている場面

### 3-5. あなたはどこから始める？

目的:

- 読者に合わせて入り口を分ける

カード案:

1. `まず何ができるか知りたい`
   遷移先: `まず体験してみる`

2. `自分に合う導入方法を知りたい`
   遷移先: `導入方法を選ぶ`

3. `料金や安全性が気になる`
   遷移先: `料金とトークン` / `セキュリティ・オプトアウト・サンドボックス`

### 3-6. 便利な活用例プレビュー

目的:

- 自分ごと化してもらう

カード例:

- ファイルを読みながら要約してもらう
- 小さな自動化スクリプトを作る
- READMEや資料のたたき台を作る
- 既存コードの理解を助けてもらう

### 3-7. 不安を下げる導線

目的:

- 初心者が読むのをやめる理由を先回りして減らす

見せる内容:

- 料金はどう考えればよい？
- オプトアウトは気にした方がよい？
- 勝手に危ない操作をされない？
- 用語が難しそうで不安

### 3-8. おすすめの読み順

目的:

- 何から読むべきか迷わせない

表示案:

1. `まず体験してみる`
2. `Chat AIとの違い`
3. `導入方法を選ぶ`
4. `はじめての使い方`

## 4. トップページで使いたい図解

### 図A. ClaudeCodeで起きること

構成:

- 人が依頼する
- ClaudeCodeが内容を読む
- ファイルを作る、または編集する
- 人が確認して次に進む

狙い:

- 「ただ答えるだけではない」を一目で伝える

### 図B. Chat AIとClaudeCodeのミニ比較図

構成:

- Chat AI: 会話中心
- ClaudeCode: 会話 + 作業 + ファイル

狙い:

- 細かい説明を読む前に差分を認識してもらう

## 5. モバイル時の見せ方

- ヒーロー図は縦並びにする
- 比較表はカード形式に変換する
- CTAは各セクションの最後にも再配置する
- おすすめの読み順は縦ステップ表示にする

## 6. コピーのトーン

- 難しさよりも「できそう」を感じさせる
- 断言しすぎず、安心感を出す
- 技術者向けすぎる単語は避ける
- ただし、便利さは具体的に言い切る

## 7. この次に必要なもの

トップページを実装に進めるには、次に以下を作るとスムーズです。

1. セクションごとの本文初稿
2. ヒーロー図と比較図のラフ
3. トップページ用のCTA文言集
