# ClaudeCode初心者向けオンボーディングサイト 「まず体験してみる」ページ初稿

## 1. このページで分かること

- ClaudeCodeが普通のChat AIとどこで違うのか
- なぜ「ファイルを作れる・直せる」が感動につながるのか
- まず何を体験すると価値が伝わりやすいのか

## 2. 3行要約

ClaudeCodeの一番分かりやすい特徴は、会話だけで終わらず、実際のファイルづくりや修正まで進められることです。  
このページでは、その感覚が伝わる小さな体験を先に見ます。  
難しい用語はあとで大丈夫です。まずは「ここまでできるんだ」を感じてください。

## 3. まずはこの体験を見てみよう

Chat AIに慣れている人ほど、ClaudeCodeを最初に見たときに驚くポイントがあります。

それは、答えを返してくれるだけではなく、実際にファイルを作ったり、ファイルを直したりできることです。

たとえば、こんなお願いをするとします。

`この内容をもとに、READMEのたたき台を作って`

普通のChat AIでも文章案は出せます。  
でもClaudeCodeでは、そこから先に進めます。

- どのファイルに入れるかを考える
- READMEとして形にする
- 必要なら修正する
- 次に足りない項目まで提案する

この「会話の結果が、実際の作業になる」という感覚が、ClaudeCodeの入口です。

## 4. 一言の指示から、ファイルができるまで

ここでは、ClaudeCodeの動きをイメージしやすいように、流れをシンプルに見てみます。

### ステップ1. 人がお願いする

例:

`このプロジェクトの説明文をわかりやすくして、README.mdを作って`

### ステップ2. ClaudeCodeが状況を読む

ClaudeCodeは、必要に応じてファイルやフォルダの中身を見て、どんな内容が必要そうかを考えます。

### ステップ3. ClaudeCodeがファイルを作る

会話の返事だけで終わらず、実際に `README.md` のようなファイルを作る、または更新するところまで進められます。

### ステップ4. 人が確認する

最後は人が確認して、「この方向でよい」「ここを直したい」と伝えながら仕上げていきます。

ここで大事なのは、全部をAIに丸投げするというより、AIと一緒に作業を進める感覚です。

## 5. 修正案だけでなく、実際に直してくれる

もうひとつ分かりやすい体験があります。

たとえば、あなたがこう思ったとします。

`この説明、ちょっと読みにくいな`

普通のChat AIなら、「こう直すとよいです」と提案してくれることが多いです。  
それだけでも便利ですが、最終的には自分で貼り直したり、反映したりする必要があります。

ClaudeCodeのよさは、その先です。

- どこが読みにくいかを見る
- 修正方針を考える
- 実際の文章を直す
- 必要なら周辺も整える

つまり、アドバイスだけで終わらず、作業を前に進めやすいのです。

## 6. 図で見る: ClaudeCodeで起きること

このページでは、次のような図を入れる想定です。

```text
あなたが依頼する
      ↓
ClaudeCodeが内容を読む
      ↓
ファイルを作る / 編集する
      ↓
あなたが確認して次に進む
```

この図で伝えたいのは、ClaudeCodeが主役になるのではなく、あなたと一緒に進む道具だということです。

## 7. 従来のやり方と比べるとどう違う？

### 従来

1. Chat AIに相談する
2. 返ってきた案を読む
3. 自分でファイルに貼る
4. 微調整する
5. 足りないところがあれば、また相談する

### ClaudeCode

1. ClaudeCodeにお願いする
2. 必要な情報を見ながら進める
3. ファイルを作る、または直す
4. あなたが確認して仕上げる

もちろん、どちらが絶対によいという話ではありません。  
でも、ファイルを実際に扱う作業では、ClaudeCodeの便利さが一気に分かりやすくなります。

## 8. 体験してみて何がすごいのか

この体験のポイントは、ただ「高機能」ということではありません。

初心者にとって大きいのは、頭の中のイメージが、すぐ形になりやすいことです。

- 思いついたことを形にしやすい
- コピペの往復が減る
- どこを直すか説明しやすい
- 作業が途中で止まりにくい

Chat AIを使っていて、

- 文章は出してくれたけど、そこから反映が面倒
- コード案はもらえたけど、どこに置くか迷う
- 修正案は分かったけど、実際に直すのが大変

と感じたことがある人ほど、この違いは刺さりやすいはずです。

## 9. まず試しやすい3つの体験

最初の体験としておすすめなのは、次の3つです。

### 1. READMEのたたき台を作ってもらう

向いている理由:

- 成果が目に見えて分かりやすい
- 失敗しても影響が小さい
- 「ファイルを作る」感覚がつかみやすい

### 2. 既存ファイルの文章を読みやすく直してもらう

向いている理由:

- 「直す」体験が分かりやすい
- ビフォーアフターを見比べやすい

### 3. 小さなスクリプトを作ってもらう

向いている理由:

- 実用感が強い
- Chat AIとの違いが伝わりやすい

## 10. 難しい言葉は、あとからで大丈夫

ここまで読んで、「セッションって何？」「ルールって？」「サンドボックスって？」と思ったかもしれません。

でも、最初は全部を理解しなくても大丈夫です。

先に大事なのは、ClaudeCodeをこう捉えることです。

`会話しながら、実際の作業にも手を伸ばせるAI`

この感覚がつかめると、あとで出てくる用語も理解しやすくなります。

## 11. 次は違いを知ろう

ここで「たしかに普通のChat AIとは違いそう」と感じたら、次は `Chat AIとの違い` に進むのがおすすめです。

次のページでは、

- 何がどう違うのか
- どんな作業で差が出やすいのか
- どんな人に向いているのか

を、比較しながら整理します。
